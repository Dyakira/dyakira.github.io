Posted on 2015-09-22 | In 读书笔记 |
《JavaScriptDOM编程艺术》（第二版）的读书笔记
  • 无论选择用双引号还是单引号，在整个脚本中保持一致
  • 最好不要使用关联数组，适用通用对象代替它
  • if语句中总是使用花括号是个好习惯
  • ==不表示严格相等（false其实和””判断是相等的）
  • ===会执行严格的比较，也就是值和变量类型的比较（!==同)
  • 定义函数时要把函数内部变量明确声明为局部变量
  • if(someting != null)与if(something)完全等价
  • 遵循“第1级DOM”能有效避免兼容性相关的问题
  • 使用this表示当前对象
  • return false 不触发行为
  • if(!method) { return false;} 如果不.请离开
  • 尽量少访问DOM尽量减少标记
  • 考虑到不支持js的情况下能否正常使用
  • js与html是标记分离的
  • 结构化程序设计原则：一个入口一个出口 多个出口只要出现在开头也是可以接受的
  • nodeName总是返回一个大写字符的值
  • 避免在body部分乱用
