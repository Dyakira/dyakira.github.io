基本配置

  1. 如何显示行号：Settings->Editor->General->Appearance标签项，勾选Show line numbers
  2. 如何设置字体：
    UI部分字体：File->Settings->Appearance&Bhavior->Appearance 中设置
    编辑区字体: File->Settings->Editor->Colors&Fonts->Font 中设置。
  3. 禁用一些不常用的插件：File -> Settings -> Plugins ->取消勾选相应的插件
  4. 安装一些常用的插件：File -> Settings -> Plugins ->搜索相应的插件
    Key promoter：提示常用鼠标点击的操作对应的快捷键..新手福利
    IdeaVim：vim爱好者福利
    MyBatis：工程用到MyBatis可以考虑.（收费）
    Jrebel：热部署（收费）
  5. 设置注释作者名字：File->Settings->File and Code Templates->Include->File Header:修改${USER}为自定义名字。
  6. 设置项目编码：File->Settings->Editor->File Encodings
    (设置默认：File->Other Settings->Default Settings-Editor->File Encodings)

配置Maven

安装Maven

去Maven官网下个安装吧..

全局配置（File–>Settings）

如下图选择maven的安装路径，idea会关联安装目录中Local respository的设置。你也可以勾选旁边的Override重新设置本地仓库的位置
注意User settings file的设置：默认是C盘下对应的目录位置。一般访问私服是需要验证信息都是在settings.xml中设置的。所以还是需要设置下的不然有些私服上的jar包无法下载到。

项目配置（Run–> Edit Configurations）

关于项目配置有几点要说明：

  1. after Launch： 可以选择项目启动后打开的浏览器
  2. On Update Action :
  • On Update 可以设置当你在Run窗口点击Update按钮的时候（或ctrl+F10）对应的行为。
  • Show dialog 勾选后会在你每次点击Update按钮的时候出现提示框。
  1. On frame Deactivation：设置窗口失去焦点后对应的操作
  • Update resources:就是除了class之外的资源会被Update
  • Update classes and resources：所有资源会被update,更改的类会被重新编译。需要注意的是类是否真的被更新取决于你的服务器设置。
    注：此处选项可能会是Hot Swap Classes,redeploy,restart server，这是因为deployment中war设置不是exploded.下面项目部署会提到
  1. 建议设置:
  • On frame deactivation->Update Resources
  • On update->Redeploy或者Restart Server
    这样可以让静态页面在窗口失去焦点快速更新，而类的话可以通过点击Update按钮的时候来更新。
    需要实现热部署的话还是建议使用JRebel等插件来实现..

项目部署

  1. 点击旁边的Deployment选项卡

  2. 添加你的项目选择Artifact类型
    注：之前并不懂Artifact是啥意思就wiki上找了下，感觉就是部署的时候生成的一种文件格式

    Artifact：one of many kinds of tangible byproducts produced during the development of software

    Artifact有两种类型

    • 普通的 .war (Web application archive) ：一种压缩格式，当部署的时候服务器才会为应用解压所需的资源
    • .war exploded 也就是没有被压缩过的格式。可以随时被改动和部署。

在这里为了调试方便我还是会选择.war exploded格式的..
